# Toxic or Nah - Content Automation Pipeline
## Complete Plan Document v2

---

## 1. OVERVIEW

### What We're Building
An automated content pipeline that produces 2 short-form videos per phone per day (TikTok + Instagram Reels) for the "Toxic or Nah" app. Each phone has a unique AI-generated girl persona with a consistent face and voice.

### Two Workflows
| Workflow | Trigger | Purpose |
|----------|---------|---------|
| **Workflow 1: Scenario Generator** | Manual / Weekly batch | Generate ContentScenario JSONs + all content copy (hook, VO, captions) |
| **Workflow 2: Daily Video Pipeline** | Daily cron (per phone) | Pick concept → get body clips → generate hook/outro → assemble final video |

### Video Structure (3 parts, multi-clip body)
```
┌──────────────────────────────────────────┐
│  HOOK (2-4s)                             │
│  Varies by concept:                      │
│  - AI girl reacting (Concept 1)          │
│  - AI couple clip (Concept 2)            │
│  - Chat screenshot (Concept 3)           │
│  - AI two girls (Concept 4)              │
├──────────────────────────────────────────┤
│  BODY (8-12s) = 3-4 clips × 2-3s each   │
│  ┌────────┬────────┬────────┬────────┐   │
│  │ Clip 1 │ Clip 2 │ Clip 3 │ Clip 4 │   │
│  │ Upload │ Score  │ Soul   │Decoded │   │
│  │ + Load │ Reveal │ Type   │Message │   │
│  └────────┴────────┴────────┴────────┘   │
│  Each clip = screen recording OR manual  │
├──────────────────────────────────────────┤
│  OUTRO (2-4s)                            │
│  Varies by concept:                      │
│  - AI girl same scene (Concept 1)        │
│  - AI girl sad/alone (Concept 2)         │
│  - None/minimal (Concept 3)              │
│  - AI two girls + result (Concept 4)     │
└──────────────────────────────────────────┘
  + Caption overlays on ALL parts
  + Voiceover audio on ALL parts
  + Background music (low volume)
```

## 2. THE 4 VIDEO CONCEPTS

### Concept 1: "Sad/Happy Girl" (`sad-happy-girl`)
> Ragazza triste/felice in casual settings

| Part | Duration | Asset |
|------|----------|-------|
| Hook | 3s | AI girl selfie-style in casual setting (bedroom, sofa). Emotion matches toxicity: sad for toxic chats, happy for wholesome |
| Body | 8-12s | 3-4 clips: app results scroll-through |
| Outro | 3s | AI girl continuation in SAME scene |

- **AI Characters**: 1 girl (phone's persona)
- **Scenarios needed**: 1
- **Environment matching**: Yes (hook + outro match body clip environment)

### Concept 2: "Before vs After" (`before-after`)
> Prima vs Dopo della Relazione

| Part | Duration | Asset |
|------|----------|-------|
| Hook | 3s | AI girl + boyfriend (couple selfie, kissing, UGC clip). Hook text: "Us before an app ruined our relationship" |
| Body | 8-12s | 3-4 clips: app results scroll-through |
| Outro | 3s | AI girl alone and sad (relationship ended after seeing results) |

- **AI Characters**: 1 girl + 1 guy (hook), 1 girl (outro)
- **Scenarios needed**: 1
- **Environment matching**: Hook = couple setting, Outro = same setting but alone

### Concept 3: "Starting from Chat" (`chat-start`)
> Video parte da una chat

| Part | Duration | Asset |
|------|----------|-------|
| Hook | 2-4s | Chat screenshot (generated by Puppeteer from scenario). No AI image needed |
| Body | 8-12s | 3-4 clips: uploads chat to app + shows results |
| Outro | 0-2s | Minimal or none (last body clip can serve as ending) |

- **AI Characters**: 0 (no image generation!)
- **Scenarios needed**: 1
- **Environment matching**: No (no AI images)
- **Cheapest concept** - no GPU costs

### Concept 4: "Comparing Exes" (`comparing-exes`)
> Girls comparing their exes archetypes

| Part | Duration | Asset |
|------|----------|-------|
| Hook | 3s | AI two girls on bed looking at phones, smiling. Hook text: "Who has the worst ex?" |
| Body | 12-18s | 3-4 clips per girl (6-8 total): Girl 1 shows her ex's score + Soul Type, then Girl 2 shows hers |
| Outro | 3s | Same 2 AI girls, follow-up scene, smiling. Outro text: "Worst Ex: Julia 95/100" |

- **AI Characters**: 2 girls
- **Scenarios needed**: 2 (one per girl's ex)
- **Environment matching**: Yes (both hook + outro match)
- **Most complex concept** - 2 scenarios, more body clips, 2 characters

### Concept Parameters Comparison

| Parameter | Concept 1 | Concept 2 | Concept 3 | Concept 4 |
|-----------|-----------|-----------|-----------|-----------|
| Hook generation | AI single girl | AI couple (girl+boy) | Puppeteer screenshot | AI two girls |
| Outro generation | AI single girl | AI girl sad | None | AI two girls |
| AI characters | 1 | 2 (hook), 1 (outro) | 0 | 2 |
| Scenarios per video | 1 | 1 | 1 | 2 |
| Body clips count | 3-4 | 3-4 | 3-4 | 6-8 (3-4 per girl) |
| GPU cost | Medium | Medium-High | Zero | High |
| Complexity | Low | Medium | Lowest | Highest |

## 3. TECH STACK

| Component | Technology | Purpose |
|-----------|-----------|---------|
| Workflow Engine | **n8n** (self-hosted) | Orchestration, scheduling, Telegram bot |
| AI Text Generation | **DeepSeek** (via n8n AI Agent) | Scenario generation, captions, VO scripts, social copy |
| Chat Screenshots | **Puppeteer** (headless browser) | Render content mode URL → screenshot (Concept 3 hook + reference) |
| AI Image Generation | **HumoAI + ComfyUI** on **SimplePod.io** | Generate consistent girl images with environment matching |
| Image-to-Video | **HumoAI + ComfyUI** on **SimplePod.io** | Convert still images to 2-4s video clips |
| Text-to-Speech | **ElevenLabs** API | Voiceover with per-phone consistent voice |
| Video Assembly | **Shotstack** API | Concat clips, add captions, mix audio |
| Database | **Airtable** | All tables: concepts, templates, scenarios, clips, runs |
| Approval | **Telegram Bot** | Inline keyboard approvals at each step |
| File Storage | **Cloudflare R2** + Airtable URLs | All generated media |

## 5. WORKFLOW 1: SCENARIO GENERATOR

### Purpose
Generate batches of scenarios with all content copy, guided by concept templates.

### Flow
```
[Manual Trigger / Schedule]
        │
        ▼
┌──────────────────────────┐
│ 1. SELECT CONCEPT        │  Random pick from active Video Concepts
│                          │  (weighted by `weight` field)
│                          │  Load linked templates:
│                          │  - Body Clip Templates (for body structure)
│                          │  - Caption Templates (for caption pattern)
│                          │  - VO Templates (for VO structure)
│                          │  - Hook Text Pool (for hook examples)
│                          │  - Social Copy Examples
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 2. AI: GENERATE          │  DeepSeek receives:
│    SCENARIO JSON         │  - Concept description + parameters
│                          │  - ContentScenario TypeScript interface
│                          │  - Valid soul type IDs from soulTypes.ts
│                          │  - Vibe (random: toxic/wholesome/mixed/decoded)
│                          │  - App style (random: imessage/instagram/whatsapp)
│                          │
│                          │  Generates N scenarios:
│                          │  - 1 scenario for Concepts 1/2/3
│                          │  - 2 scenarios for Concept 4 (one per girl)
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 3. VALIDATE              │  Parse JSON against ContentScenario type
│                          │  Check soul type IDs exist
│                          │  Ensure scores 0-100
│                          │  Ensure 3-6 messageInsights with tag mix
│                          │  No "Early Stage" traits
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 4. AI: GENERATE          │  DeepSeek receives scenario + all templates:
│    CONTENT COPY          │
│                          │  Generates:
│                          │  a) Hook text (picks from pool or generates new)
│                          │  b) Outro CTA text
│                          │  c) VO script (following VO Template structure,
│                          │     one section per body clip slot)
│                          │  d) Caption plan (following Caption Template
│                          │     pattern, concrete text filled in)
│                          │  e) Social media caption + hashtags
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 5. TELEGRAM APPROVAL     │  Send summary:
│                          │  - Concept name + vibe
│                          │  - Chat preview (first/last 3 messages)
│                          │  - Toxicity score
│                          │  - Hook text + VO script preview
│                          │  [✅ Approve] [🔄 Regenerate] [❌ Skip]
└──────────┬───────────────┘
           │ (on approve)
           ▼
┌──────────────────────────┐
│ 6. SAVE TO AIRTABLE      │  Create row(s) in Scenarios table
│                          │  Status: "approved" → "clips_needed"
│                          │
│ 7. DEPLOY TO CONTENT     │  Save JSON to public/scenarios/{name}.json
│    MODE                  │  (for body clip recording on Sunday)
└──────────────────────────┘
```

### Batch Mode
- Run generates scenarios for the week ahead
- Per phone: `videos_per_day × 7` = 14 scenarios/week
- 3 phones × 14 = **42 scenarios/week**
- Concept distribution based on `weight` field in Video Concepts table
- Vibe mix: ~40% toxic, ~25% mixed, ~20% decoded, ~15% wholesome

## 6. BODY CLIP RECORDING (Sunday Workflow)

### Two Recording Modes

#### Mode A: Screen Recording (Automated)
```
For each scenario with clips_needed status:
  1. n8n launches Puppeteer
  2. Opens content mode: https://content.toxicornah.com/?scenario={name}
  3. App loads scenario, shows results
  4. For each clip slot in Body Clip Template:
     - Navigate to the app_section (score_reveal, soul_type_card, etc.)
     - Record screen for min-max duration
     - Save clip to Cloudflare R2
  5. Upload clip URLs to Body Clips table
```

#### Mode B: Manual Recording (UGC)
```
Sunday bulk recording session:
  1. User opens Airtable → filters Body Clips by status=pending
  2. For each clip:
     - Opens scenario URL on phone
     - Navigates to the section specified in clip_label
     - Records themselves scrolling/reacting
     - Uploads clip to Airtable row
  3. Pipeline detects upload → status: "uploaded" → "ready"
```

#### Mode Selection Logic
```
if concept.body_screen_recording_enabled AND concept.body_manual_recording_enabled:
    mode = random.choice(["screen_recording", "manual"])
elif concept.body_screen_recording_enabled:
    mode = "screen_recording"
elif concept.body_manual_recording_enabled:
    mode = "manual"
```

### Environment Frame Extraction
After body clips are ready (regardless of mode):
1. Take the **first body clip** of the set
2. FFmpeg: extract frame at t=1s → `environment_frame.jpg`
3. Vision AI describes the environment
4. Save frame + description to Body Clips row
5. This is used for hook/outro AI image generation (environment matching)

## 18. KEY DECISIONS SUMMARY

| Decision | Choice | Reason |
|----------|--------|--------|
| Architecture | Concept-based with templates | Same pattern as marble business, flexible per-concept parameters |
| Body clips | Multi-clip (3-4 per concept) with templates | Each clip shows different app section, more dynamic |
| Recording mode | Checkbox per concept (screen/manual/both) | Flexibility + random selection for variety |
| Chat screenshots | Puppeteer headless browser | Reuses existing React components |
| Video assembly | Shotstack API | Cloud-based, handles multi-clip concat + overlays |
| AI image gen | HumoAI + ComfyUI on SimplePod.io | LoRA face consistency + IP-Adapter env matching |
| Text generation | DeepSeek | Cheap, good quality, n8n AI Agent compatible |
| TTS | ElevenLabs | Best quality voices |
| Database | Airtable (13 tables) | Visual UI, easy n8n integration, template pattern |
| Approval | Telegram Bot | Mobile-friendly, inline buttons |
| Storage | Cloudflare R2 | Cheap, S3-compatible |
| Overlay text | Post-production (Shotstack) | Not burned into AI images |
| Phones | 3 with active toggle | Gradual scaling |
